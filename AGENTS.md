# Cordova AutoArcGIS — Agent Instructions

## Project Overview
Spike project integrating [Apache Cordova](https://cordova.apache.org/) with ArcGIS on Android. The app shell is a standard Cordova scaffold; the goal is to embed ArcGIS Maps SDK (or ArcGIS JS API via WebView) inside the Cordova WebView.

- **Platform:** Android only (`cordova-android ^15.0.0`)
- **App ID:** `com.example.autoarcgis`
- **Min SDK:** 24 | **Target SDK:** 36

## Key Directories

| Path | Purpose |
|------|---------|
| `www/` | Web source — edit here, not in `platforms/` |
| `www/js/index.js` | Entry point; wires up `deviceready` event |
| `www/index.html` | Main HTML; CSP header must be updated to allow ArcGIS CDN |
| `config.xml` | Cordova project configuration (plugins, preferences, permissions) |
| `platforms/android/` | Generated — do not edit directly; changes are overwritten by `cordova prepare` |
| `platforms/android/cdv-gradle-config.json` | Gradle/SDK versions reference |

## Build & Run Commands

```bash
# Add/restore Android platform (first time or after clean)
npx cordova platform add android

# Build debug APK
npx cordova build android

# Run on connected device or emulator
npx cordova run android

# Copy www/ changes into platforms/ without rebuilding native code
npx cordova prepare android
```

> Requires Android SDK (API 36) and a JDK 11+ on PATH. `ANDROID_HOME` env var must point to the SDK root.

## Cordova Conventions

- **All web code lives in `www/`** — `platforms/android/app/src/main/assets/www/` is generated by `cordova prepare`.
- Plugins are declared in `config.xml` and installed via `cordova plugin add <id>`. Installed plugin source goes into `plugins/` (not yet present).
- The `deviceready` event must fire before using any Cordova or native API.
- `cordova.js` is injected at build time; never add it to `www/`.

## ArcGIS Integration Notes

- **ArcGIS Maps JS API** (CDN approach): update the CSP in `www/index.html` to allow `https://js.arcgis.com` and `https://*.arcgis.com`.
- **ArcGIS Maps SDK for Android** (native approach): requires adding Maven repo (`https://esri.github.io/arcgis-gradle-plugins/`) and a custom Cordova plugin or native bridge.
- Prefer the JS API via CDN for a pure WebView approach; use the native SDK only when offline maps or advanced native features are needed.

## Content Security Policy
Current CSP in `www/index.html` must be extended before any ArcGIS CDN calls will work:
```
default-src 'self' data: https://ssl.gstatic.com https://js.arcgis.com https://*.arcgis.com 'unsafe-eval';
```

## Pitfalls
- Editing files under `platforms/` directly is lost on the next `cordova prepare` or `cordova build`.
- `cordova-android 15` requires Gradle 8+ and AGP 8+; see `platforms/android/cdv-gradle-config.json` for exact versions.
- Java source/target compatibility is set to 11; do not introduce Java 17+ features in any custom native code without updating `cdv-gradle-config.json`.
